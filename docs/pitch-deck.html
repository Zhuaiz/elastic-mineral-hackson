看一块石头，

一个用 Elasticsearch 融合检索（RRF）驱动的矿物鉴定 Agent —— 照片、属性、硬度，一次查询融合定种。

jina-clip-v2 图文同空间 · 阿里云 ES 9.3.2 · Agent Builder · trapstreet 公开评测
Nº 02THE PROBLEM
为什么矿物鉴定难

单一模态，都不够。

只看照片 → 混淆
玛瑙 · 玉髓 · 碧玉
都是隐晶质石英，肉眼与图像模型都难分。
只看属性 → 不收敛
硬度7 · 白条痕 · 玻璃光泽
被几十种矿物共享，定不到唯一种。

几千种矿物，很多长得几乎一样、属性也高度重叠。任何单一信号都不足以定种。

Nº 03THE IDEA
我们的答案

把三种信号
融合进一次检索。

每个标本在 Elasticsearch 里同时可被三种方式检索——图像向量、属性全文、结构化字段。 ES 9.3 的 rrf retriever 在一个查询里把三路做倒数排名融合，9.3 原生自带这项能力。

Nº 04ARCHITECTURE · 三层
怎么搭的
01

数据层

MineralImage5k 近 5,500 张标本照片 · 98 个矿物种 · Mindat 结构化属性（硬度/条痕/晶系/密度）· Wikipedia 描述 → Elasticsearch

02

检索层 · 核心

图像 kNN（jina-clip-v2）+ 属性 BM25 + 硬度/晶系结构化过滤 → RRF 一次融合

03

Agent 层

Agent Builder 的 mineralogist：ES|QL 属性工具 + 融合检索工具，agent 自主编排；支持中文提问

Nº 05THE CORE · retriever.rrf
融合检索长什么样
kNN 图像向量 · jina-clip-v2
BM25 属性全文 · 条痕/颜色/光泽
filter 硬度范围 · 晶系
→
RRF
one query · one ranking
候选矿物
按种名投票定种

图和文在同一个 1024 维空间——文字能搜图，图也能搜图。三路信号互补，倒数排名融合成一个结果。

Nº 06AGENT BUILDER
Kibana 里直接搭 agent

mineralogist agent
自己决定怎么查。

我给它两个 ES|QL 属性工具和一个融合检索工具，它自主编排：先按硬度筛，还是先按图搜。

中文提问 →
"绿色带条带、条痕也绿、硬度约4，是什么矿物？"
Agent 作答 →

附匹配证据 + 备选，并说明如何区分。jina-clip 文本塔支持 89 种语言。

Nº 08PROOF · 公开可复现
作答准确率 vs 检索强度

检索越强，作答越准。

48%
