# 演讲稿 · 矿物鉴定 Agent（4 分钟 pitch）

> 目标：让评委在 4 分钟内记住一件事——**"细粒度矿物鉴定，单一模态搜不准，Elastic 融合检索才行，而且我们用数字证明了。"**

## 开场（30 秒）— 用一个真问题抓人

> "地质队员在野外捡到一块石头，想知道它是什么矿物。他有一张照片、几个肉眼可观察的属性——颜色、条痕、硬度。
> 但矿物鉴定是出了名的难：几千种矿物，很多长得几乎一样。玛瑙、玉髓、碧玉都是隐晶质石英，**光看照片会混**；
> 而'硬度7、白条痕、玻璃光泽'又被几十种矿物共享，**光看属性也定不了**。
> 我们做的，是一个用 Elastic 融合检索来解决这个'单模态都不够'问题的矿物鉴定 Agent。"

## 设计怎么讲（90 秒）— 三层，一句一层

**1. 数据层（一句带过）**
> "我们把 MineralImage5k 的近 5,500 张标本照片、98 个矿物种，连同从 Mindat 抓来的结构化属性（硬度、条痕、晶系、密度）和 Wikipedia 描述，一起进了 Elasticsearch。"

**2. 检索层（核心，讲透 RRF）**
> "关键在检索。每张标本在 ES 里同时有三种可检索信号：
> 图像向量（jina-clip-v2，图和文在**同一个 1024 维空间**）、属性全文（BM25）、结构化字段（硬度、晶系可做范围和精确过滤）。
> ES 9.3 的 `rrf` retriever 把这三路**在一个查询里融合**——图像 kNN + 属性 BM25 + 硬度过滤，一次调用，倒数排名融合。
> 这不是我们拼的，是 Elastic 原生能力。"

**3. Agent 层（体现 Elastic 生态）**
> "上面是 Agent Builder 构建的 mineralogist agent：我给它两个 ES|QL 属性工具和一个融合检索工具，
> 它自己决定先按硬度筛、还是先按图搜。中文提问也行，因为 jina-clip 文本塔支持 89 种语言。"

## 演示（60 秒）— 让屏幕说话
> "看实物——"（切演示界面，拖一张照片）
> "三栏并排：只看图、只看字、RRF 融合。只看图，玛瑙玉髓碧玉混在一起；
> 加上属性和硬度过滤，融合这一栏第一名立刻锁定。**这一栏赢，就是 RRF 的价值。**"
> （再切 Kibana Agent Builder，中文问一句，展示 agent 自动调工具作答。）

## 证明（30 秒）— 我们不自说自话
> "怎么证明融合真的更好、不是我嘴上说？我们把它做成了 trapstreet 上的公开评测：
> 同一套 50 道鉴定题、同一个判分器，只改'喂给模型的检索证据'——
> 闭卷、单路、到 RRF 融合，**作答准确率一路抬升**。检索配置是变量，判分准确率是因变量，曲线爬升就是结论。
> 每个数字都是真实的一次运行，公开可复现。"

## 收尾（20 秒）— 落到价值
> "地质勘探、宝石鉴定、地质教育、野外科考——凡是'看到一块石头想知道它是什么'的场景，
> 这套融合检索 + Agent 的范式都能用。谢谢。"

## 评委可能问的（Q&A 准备）

**Q: 纯用多模态大模型（GPT-4V/Qwen-VL）直接看图不就行了？**
> 我们的闭卷基线就测了这个——强模型看图对细粒度矿物 top-1 很低（视觉歧义 + 需要专业先验）。
> RAG 把它接到一个**结构化的、可过滤的、领域专属**的矿物库上，准确率显著上去。而且可解释：给出匹配了哪些属性。

**Q: 为什么要 RRF，纯向量 kNN 不够吗？**
> 图像 kNN 会被视觉相似性带偏（金和黄铁矿、玛瑙和玉髓）。属性 BM25 和硬度过滤提供正交信号。
> 消融表里 RRF 明显高于任一单路——这正是我们要证明的。

**Q: jina-clip-v2 是非商业许可？**
> 是 CC-BY-NC，黑客松演示没问题，已在 NOTICE 标注。要商业化就换 EIS 托管的 Jina 或 Apache-2.0 的 SigLIP2，架构不变。

**Q: 数据够吗 / 泛化性？**
> 98 种是 demo 规模，架构对更大种类线性扩展（Mindat 有 6000+ IMA 种）。图片是 Fersman 博物馆标本，仅演示用、不再分发。

**Q: 用的是阿里云还是 Elastic Cloud？**
> 阿里云 ES 9.3.2（黑客松方案一），license 是 enterprise，RRF / Agent Builder / Workflows 全可用。
> 唯一 EIS 不在阿里云，所以图像嵌入我们在本地算好写入 dense_vector——反而更省事、更可控。

## 一句话记忆点（如果只能说一句）
> **"单一模态搜不准细粒度矿物；Elastic 的 RRF 融合检索把图像、属性、硬度过滤合到一个查询里，作答准确率随融合强度上升——我们用公开可复现的数字证明了。"**
